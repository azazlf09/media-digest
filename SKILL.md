# MediaDigest Skill v3.0

Universal video content summarization pipeline. Supports YouTube, Bilibili, and X/Twitter. Users provide a video link and get an AI-generated transcript + summary. Also supports channel monitoring for YouTube and Bilibili.

## When to Use

- User sends a video URL (YouTube, Bilibili, X/Twitter) and wants a summary
- User wants to monitor YouTube/Bilibili channels for new content
- User asks about video content, transcript extraction, or channel updates

## Supported Platforms

| Platform | Download | Transcribe | Channel Monitor | Auth Required |
|----------|----------|------------|-----------------|---------------|
| YouTube | Yes | Yes (subs + Whisper) | Yes | No (most videos) |
| Bilibili | Yes | Yes (Whisper) | Yes | No (audio) |
| X/Twitter | Yes | Yes (Whisper) | No (manual only) | Yes (cookies) |

## Quick Start

### 1. Install Dependencies

```bash
bash skills/media-digest/setup.sh
```

### 2. Process a Single Video

```bash
python3 skills/media-digest/media_digest.py now <VIDEO_URL>
```

The agent should:
1. Run the command above
2. Read the output summary JSON from `data/summaries/`
3. Present the transcript to the user in a structured format
4. If the user wants a summary, use the transcript as context for AI summarization

### 3. Channel Monitoring

```bash
# Add a channel
python3 skills/media-digest/media_digest.py add <CHANNEL_URL> [alias]

# Check for new videos
python3 skills/media-digest/media_digest.py check [count]

# List channels
python3 skills/media-digest/media_digest.py list
```

## AI Agent Workflow

### When a user sends a video URL:

1. **Detect platform** — Identify YouTube, Bilibili, or X/Twitter
2. **Run processing**:
   ```bash
   python3 skills/media-digest/media_digest.py now <url>
   ```
3. **Read the output** — The summary JSON will be in `data/summaries/<platform>_<video_id>.json`
4. **Present to user** — Format the transcript with:
   - Title and author
   - Duration and language
   - Key points / summary (generated by AI from transcript)
   - Original URL

### When checking channels:

1. Run `python3 skills/media-digest/media_digest.py check 5`
2. If new videos found, read each summary JSON
3. Report each new video's title and brief summary to the user
4. If no new videos, report "All channels up to date"

### If processing fails with auth error:

1. Tell the user: "This video requires authentication. For local users, make sure you're logged into the platform in your browser. For Docker users, cookies files need to be placed in `data/cookies/<platform>/cookies.txt`."
2. Provide the specific error message from the tool output

## All Commands

| Command | Description |
|---------|-------------|
| `now <url>` | Process a single video immediately |
| `add <url> [alias]` | Add a channel to monitor |
| `remove <id>` | Remove a channel |
| `list` | List monitored channels |
| `check [count]` | Check channels for new videos (default: 3) |
| `latest [count]` | Show latest N summaries |
| `report` | Generate text report of all summaries |
| `deps` | Check dependency status |
| `migrate <v2_dir>` | Migrate v2.0 channel-monitor data |

## Output Format

Each processed video produces `data/summaries/<platform>_<video_id>.json`:

```json
{
  "video_id": "BV1GJ411x7h7",
  "platform": "bilibili",
  "title": "Video Title",
  "uploader": "Author Name",
  "duration": 180,
  "upload_date": "2019-12-31",
  "language": "zh",
  "transcription_method": "whisper",
  "transcript_length": 1880,
  "transcript": "Full transcript text...",
  "url": "https://www.bilibili.com/video/BV1GJ411x7h7",
  "processed_at": "2026-03-19T01:00:00Z"
}
```

## Environment Variables

- `MEDIA_DIGEST_DIR` — Data directory (default: `skills/media-digest/data/`)
- `MEDIA_DIGEST_WHISPER_MODEL` — Whisper model size (default: `base`)

## Graceful Degradation

- **No Whisper installed**: Still downloads and returns metadata + subtitles (if available)
- **No cookies**: Returns actionable error with setup instructions
- **No ffmpeg**: Returns error with install command
- **Rate limits**: Processes oldest-first, respects platform limits

## Data Storage
